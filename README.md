# 欢迎来到《AI原生软件开发》课程仓库

很高兴与你一起开启这门课程。

这不是一门只讲概念的课，而是一门强调**边学边做、以做促学**的实战课程。在接下来的 8 周里，我们会围绕 AI 原生软件开发的真实工作流，逐步完成从提示工程、编码智能体开发、AI IDE 集成，到自动化、安全合规、代码审查，再到端到端应用构建与运维观测的完整训练。

希望你在这里不仅学会“怎么用 AI 写代码”，更能建立起一套面向真实项目的思维方式：

- 会提出更清晰的问题
- 会设计更可靠的人机协作流程
- 会用测试、文档和工程规范约束 AI
- 会把 AI 真正变成提升开发效率的生产力工具

## 课程信息

- **课程名称**：AI原生软件开发
- **培训周期**：8 周
- **上课频次**：每周 1 次，每次半天（3 小时）
- **培训形式**：线下授课，采用“理论讲解 + 案例演示 + 场景实操”三段式教学
- **适合对象**：具备基础软件开发能力，了解常用开发工具与流程的技术人员
- **实验环境**：基于 TRAE 大模型开发平台，核心模型使用 Kimi K2.5

## 你将学到什么

本课程将围绕 AI 融合开发的完整链路展开，重点帮助大家提升以下能力：

- 理解 LLM 的基本原理与常见失效模式
- 掌握结构化提示词设计与效果调优方法
- 理解 coding Agent 架构、工具调用与 MCP 集成方式
- 学会在 AI IDE 中进行代码开发与知识库协同
- 建立可复用的自动化开发流程与多 Agent 协作方式
- 掌握 AI 生成代码的测试、安全与合规控制方法
- 学会对 AI 生成内容进行代码审查、调试与文档维护
- 能独立完成一个端到端 AI 原生应用的构建与交付

## 课程安排一览

### Week 1：LLM 基础与高效提示工程
聚焦大模型基础原理、提示工程方法，以及如何在开发任务中设计高质量提示词。

### Week 2：编码智能体核心架构与最小原型实现
围绕 coding Agent 架构、工具调用与最小应用改造，完成一个可运行、可测试、可维护的原型升级。

### Week 3：AI IDE 与外部系统集成
学习 AI IDE 的上下文管理、Spec 编写，以及企业知识库与开发工具的集成方式。

### Week 4：编码智能体管理与人机协作模式
理解不同自治级别的编码智能体如何落地，并建立人审点、安全护栏与工程协作机制。

### Week 5：AI 增强终端与开发流程自动化
通过脚本、工作流资产与多代理协作，把高频重复任务沉淀为可复用的开发流程。

### Week 6：AI 生成代码的测试与安全合规
基于安全扫描与修复闭环，学习如何在 AI 开发中落实安全工程与合规要求。

### Week 7：AI 辅助代码审查、调试与智能文档生成
训练“AI 先写，人来审”的工作模式，提升代码 review、调试与文档维护能力。

### Week 8：端到端 AI 原生应用构建与运维观测
完成跨技术栈的应用交付，建立从开发到运行、观测与复盘的完整视角。

## 建议的学习方式

为了更好地完成这门课程，建议你采用以下节奏：

1. **先理解任务目标**：明确本周要解决什么问题、验收标准是什么。
2. **再使用 AI 辅助实现**：不要一上来就把问题全丢给模型，先想清楚再提问。
3. **保留过程证据**：记录提示词、关键决策、测试结果与踩坑过程。
4. **重视可复现性**：确保代码、命令、文档可以被他人顺利复现。
5. **坚持人工复核**：AI 可以加速，但最终质量要靠你把关。
